# Level 2 - Task 3: To-Do Web App

An interactive, responsive, and beautifully styled **To-Do Web App** that enables users to manage daily objectives. Developed as part of the **OASIS Infobyte Summer Internship Program (OIBSIP)**.

## Project Demo
- **Live Preview:** [View Project](https://vaishnavithorat6852.github.io/-OIBSIP/OIBSIP/WebDevelopment-Level2-Task3-ToDoWebApp/)

## Features
- **Dynamic CRUD State**: Add, edit, complete, and delete tasks dynamically with state-driven rendering.
- **Lists Segregation**: Automatically sorts tasks into "Pending Tasks" and "Completed Tasks" columns.
- **Progress Badges**: Count badges ("X pending", "Y completed") that update dynamically as items change state.
- **Inline Editing Mode**: Transform task descriptions into editable text inputs inline and click "Save" to commit the changes.
- **Bonus Feature: Timestamps**: Displays exact date and time indicators for when a task was added and when it was completed.
- **Bonus Feature: Local Storage Persistence**: State is stored in the browser's `localStorage`, maintaining data across page refreshes.
- **Custom Animations**: Slide-in transitions on task creation, fade-out transitions on task removal, and smooth checks.
- **Responsive Layout**: Adapts from a parallel grid on desktop to stacked blocks on mobile screen viewports.

## Tech Stack
- **HTML5**: Structured input cards and layout columns.
- **CSS3**: Glassmorphic styling card properties, grid system alignment, and transition animations.
- **Vanilla JavaScript**: Pure state-based CRUD architecture and browser local storage sync.

## Running Locally
1. Clone this repository:
   ```bash
   git clone https://github.com/vaishnavithorat6852/-OIBSIP.git
   ```
2. Navigate to the project directory and open `index.html` in a web browser:
   ```bash
   cd -OIBSIP/OIBSIP/WebDevelopment-Level2-Task3-ToDoWebApp
   python -m http.server 8000
   ```
3. Open `http://localhost:8000` in your web browser.
